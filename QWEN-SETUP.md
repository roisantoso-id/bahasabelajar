# 接入 Qwen（翻译 + TTS）+ 防盗刷

前端已接好：**默认走 Qwen，连不上/超限自动回退**（翻译→MyMemory，发音→系统语音），所以线上不会坏。
部署好下面的东西后，Qwen 自动生效。

> 为什么要边缘函数：静态站浏览器直连百炼被 CORS 拦，且 key 不能进公开页面。函数把 key 存服务端、解决 CORS、并做防盗刷。

## 防盗刷三层（重点）

1. **必须登录**：函数会校验调用者的登录 token，没登录直接拒。挡住网上随便扫到 URL 的人。
2. **每用户每日上限**：默认每人每天 300 次，超了返回 429。挡住注册账号后狂刷。
3. **百炼预算上限**：在阿里云/百炼控制台给账号或这个 key 设**消费上限/告警**——无论如何花不超这个数（强烈建议设，这是最后兜底）。

---

## 部署步骤

### 第 1 步 · 建用量表
SQL Editor 运行 **`supabase/qwen-usage.sql`**（建 `qwen_usage` 表，只有函数能写）。

### 第 2 步 · 建函数
Dashboard → **Edge Functions** → Create function
- 名字必须叫 **`qwen`**
- 粘贴 **`supabase/functions/qwen/index.ts`** 全部内容
- **"Verify JWT" 关掉**（身份由函数内部手动校验，避免 CORS 预检被平台拦）
- Deploy

### 第 3 步 · 设密钥
Edge Functions → **Secrets** → 新增：
- `DASHSCOPE_KEY` = 你给我的 `sk-ws-...` key
- （可选）`QWEN_DAILY_CAP` = 想要的每人每日上限，比如 `200`（不设默认 300）

> `SUPABASE_URL`、`SUPABASE_SERVICE_ROLE_KEY` 是平台自带，**不用你设**。

### 第 4 步（强烈建议）· 百炼设预算上限
阿里云百炼/Model Studio 控制台 → 费用/额度 → 给账号或该 API key 设**消费上限**或**用量告警**。
这样即使前两层被绕过，花费也有硬顶。

### 第 5 步 · 告诉我「好了」
我会从预览直接打函数验证：①未登录被拒（401）②登录后翻译返回中文、TTS 返回音频 ③划词弹窗标签变「Qwen 翻译」。

---

## 额外可选的收紧
- **关闭开放注册**：Dashboard → Authentication → 关掉 Sign-ups（或限制）。这样只有你/你邀请的人能注册，第 1 层就基本等于「只有你能用」。适合纯自用。
- **调低每日上限**：把 `QWEN_DAILY_CAP` 设小（如 100），日常够用又更稳。

## 用法 & 费用
- **翻译**：词库没有的词用 `qwen-mt-turbo`（比 MyMemory 好很多）；词库有的仍显示词库释义。
- **发音**：默认 Qwen；**我的 → 发音引擎** 一键切「系统」（免费/离线/真印尼语）。⚠️ Qwen TTS 不支持印尼语（用 auto，可能不准）。
- 翻译很便宜，TTS 较贵。不想花钱就把发音切「系统」，翻译影响很小。
