## 第 1 步 · Dashboard 关闭邮箱确认（不是 SQL）

Supabase 控制台 → **Authentication** → **Sign In / Providers** → **Email** →
把 **"Confirm email"（确认邮箱）关掉** → Save。

> 为什么：默认注册要去邮箱点确认链接才能登录。个人 app 关掉它，注册完直接就能用。

## 第 2 步 · 跑一段 SQL

SQL Editor → 新建查询 → 把 **`supabase/auth-notes-migration.sql`** 整个文件粘进去 → Run。

它会做：清空旧的学习进度/打卡（你选了「清空重来」）、给进度表加 user_id、建 `profiles`、`article_notes` 表、设好按账号隔离的 RLS。
（词库 / 文章 / 语法 / 文化 都不动，每日 routine 照常。）

## 这次新增/改动一览

- **登录页**：没登录进不来；数据按账号隔离（真多用户）。
- **底部导航**改为：今日 · 文章 · 词库 · **日历** · **我的**（文化挪到「今日」页的一个入口）。
- **日历页**：月历 + 连续学习天数 + 打卡标记（仿你小程序那版）。
- **我的页**：头像/邮箱、已学/连续/笔记统计、每日新词&复习上限可调、退出登录、Supabase 配置（折叠在高级里）。

- 顺手修了一个老 bug：打卡的「完成」状态以前从不置位，导致连续天数一直 0；现在学满当天目标就算完成打卡，日历/连续天数才有意义。
